KNOWN ISSUES COVERED
====================

1. asyncpg vs psycopg2 dialect
   Location: database.md - "Connection string must be a string or URL instance"
   Fix: DATABASE_URL must use postgresql+asyncpg:// prefix

2. JWT_SECRET ordering
   Location: service-startup.md, authentication.md, tests.md
   Fix: Set JWT_SECRET before importing any app modules

3. python-multipart missing
   Location: service-startup.md
   Fix: Add to requirements.txt, rebuild container

4. Docker image caching
   Location: docker-deployment.md
   Fix: Use docker compose down --rmi local && docker compose up -d --build

5. git pull blocked by untracked files
   Location: docker-deployment.md
   Fix: Use git clean -f <file> then git pull

6. PyJWT not found in container
   Location: service-startup.md
   Fix: Dockerfile COPY order - requirements.txt to /tmp before COPY .

USAGE PATTERNS
==============

Pattern 1: Start with symptom
1. Read docs/troubleshooting/README.md
2. Find symptom category
3. Jump to relevant guide (service-startup.md, database.md, etc.)
4. Apply fix
5. Verify with health checks from QUICK_REFERENCE.md

Pattern 2: Quick command lookup
1. Go to QUICK_REFERENCE.md
2. Find command in relevant section
3. Copy and adapt to your situation
4. Run command

Pattern 3: Deep investigation
1. Start with investigation workflow in README.md
2. Follow 5-step process
3. Reference specific issues in detailed guides
4. Use debugging checklists

Pattern 4: CI/CD failures
1. Check test failures in tests.md
2. Verify environment variables set (docker-deployment.md)
3. Ensure dependencies running (database.md, service-startup.md)

FORMAT & STYLE
==============

Each issue follows this structure:
1. Error message or symptom (heading)
2. Symptom description (what you see)
3. Root cause explanation (why it happens)
4. Immediate fix (step-by-step resolution)
5. Prevention (how to avoid in future)
6. Optional: Extra debugging steps

Format features:
- Exact bash commands (copy-paste ready)
- Real error messages (easy grep/search)
- Prevention with code examples
- Cross-references to related guides
- Debugging checklists
