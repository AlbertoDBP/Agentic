================================================================================
                          KEY INFORMATION CAPTURED
================================================================================

SERVICE OWNERSHIP MODEL:
  • Agent 01 (Market Data) → price_history, market_data_daily
  • Agent 02 (Newsletter) → analysts, articles, recommendations, accuracy log
  • Agent 03 (Scoring) → scoring_runs, quality_gate_results, income_scores
  • Agent 04 (Classification) → asset classifications, rules, overrides
  • Agent 05 (Tax) → (stateless, no DB writes)
  • Agent 06 (Simulation) → scenario_results
  • NAV Service → covered_call_etf_metrics, erosion cache

DATA FLOW ORDER:
  1. Agent 01 fetches market data (price_history, market_data_daily)
  2. Agent 04 classifies assets (asset_classifications)
  3. Agent 03 runs quality gates and scores (income_scores)
  4. Agent 02 ingests analyst intelligence (continuous)
  5. Agents 05, 06 consume data on-demand (no writes)

CACHE VALIDITY:
  • Quality gates: 24h
  • Income scores: 24h
  • Classifications: 30 days
  • Analyst recommendations: 90 days (auto-expire)
  • NAV erosion cache: 7 days

CROSS-TABLE DEPENDENCIES:
  • income_scores depends on quality_gate_results
  • income_scores depends on nav_erosion_analysis_cache
  • asset_classifications consumed by agents 03, 05, 06
  • analyst_recommendations consumed by dashboards and Agent 12

INDEXING STRATEGY:
  • Single-column: ticker, ticker_symbol, symbol, analyst_id, created_at
  • Composite: (ticker, scored_at DESC), (ticker, is_active, decay_weight DESC)
  • Vector: IVFFlat on embedding columns for semantic search
  • All indexes documented with use cases
